Análise exploratória

O que os dados revelam

Antes de modelar, é preciso entender. A EDA revelou os fatores que mais influenciaram a sobrevivência.

♀
Sexo é a feature mais importanteMulheres sobreviveram 74% das vezes; homens apenas 19%. A regra "mulheres e crianças primeiro" foi real.
🎩
Classe social importa muito1ª classe: 63% de sobrevivência. 2ª: 47%. 3ª classe (maioria): apenas 24%.
👶
Crianças tiveram prioridadeMenores de 12 anos tiveram ~58% de taxa de sobrevivência. Discretizar Age em faixas captura esse padrão.
💰
Tarifa alta = mais chancesPassageiros no quartil mais alto sobreviveram em ~65% dos casos. Distribuição altamente assimétrica.

Pré-processamento

Pipeline customizado

Seis transformadores encadeados, todos compatíveis com Pipeline, GridSearchCV e cross_val_score.

01
ColumnDropper
Remove Cabin (~77% nulo), Name e Ticket — colunas sem valor preditivo direto.
errors='ignore'
02
CategorizerFeatures
Age → 7 faixas etárias + "Não_Informado" para nulos. Fare → 4 quartis. Elimina outliers e preserves o sinal da ausência.
fit() aprende fare_max_ do treino — sem data leakage
03
OrdinalEncoderTransformer
Codifica Sex ordinalmente: female → 0.0 | male → 1.0.
sklearn.OrdinalEncoder
04
OneHotEmbarkedAndPclassEncoder
One-Hot encoding para Embarked, Pclass, faixa_etaria e Fare_categoria. handle_unknown='ignore' para categorias novas.
sparse_output=False
05
SimpleImputer
Preenche nulos remanescentes com a moda. Cuida dos 2 nulos em Embarked.
strategy='most_frequent'
06
StandardScaler
Normaliza features numéricas (SibSp, Parch, Sex) para média 0 e desvio-padrão 1.
parâmetros aprendidos apenas no treino

Modelagem

Três modelos, um ensemble

Cada modelo foi otimizado individualmente com GridSearchCV (cv=5). O ensemble combina os três para maximizar a generalização.

📐
Regressão Logística
Notebook 02 · baseline interpretável
Modelo linear com regularização. Rápido, interpretável e coeficientes revelam a importância de cada feature.
C1
penalty'l2'
solver'liblinear'
🌲
Random Forest
Notebook 03 · 75 árvores, bootstrap
Ensemble de árvores robusto a outliers. Captura não-linearidades e fornece importância de features.
n_estimators75
min_samples_leaf2
min_samples_split10
⭐ Final
🗳
Voting Classifier
Notebook 04 · Soft Voting · LR + RF + KNN
Combina os três modelos pela média das probabilidades (Soft Voting). Reduz variância ao combinar erros distintos.
voting'soft'
estimadoreslr · rf · knn
knn_metric'manhattan'

Instruções

Como executar

Em 5 passos, do clone ao Streamlit rodando no seu navegador.

1
Clone o repositório
Faça o clone do projeto para sua máquina local.
git clone https://github.com/Caetanodassis/titanic_ML.git cd titanic_ML
2
Crie e ative o ambiente virtual
Isole as dependências do projeto.
# No Windows: python -m venv venv venv\Scripts\activate # No Linux/Mac: python3 -m venv venv source venv/bin/activate
3
Instale as dependências
Instale todas as bibliotecas necessárias listadas no requirements.txt.
pip install -r requirements.txt
4
Treine o modelo (Opcional)
Gere o arquivo do pipeline serializado se ele não existir.
python treinar_modelo.py
5
Execute o Streamlit
Rode o servidor local do Streamlit utilizando o arquivo de interface.
streamlit run interface.py
